剑指offer 66题个人题解

1. 二维数组查找

在一个二维数组中（每个一维数组的长度相同），每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。leetcode 74
方法一：暴力枚举 row major order 遍历 O(mn)效率很差
方法二：从右上角开始step search每一步比当前元素小就往左走 比当前元素大就往下走
step search on sorted matrix worst case O(m+n) 需要行内严格排序 行与行之间可以不排序
例如

特殊情况：若矩阵严格row major order排序，则可以使用binary search O(lg(m+n))
例如
[1, 3, 5, 7]
[10, 11, 16, 20]
[23, 30, 34, 50]
binary search on 2d fully sorted row major order matrix

2. 替换空格

请实现一个函数，将一个字符串中的每个空格替换成“%20”。例如，当字符串为We Are Happy.则经过替换之后的字符串为We%20Are%20Happy
方法一：直接遍历替换，每一次遇到空格替换为%20并将后续字符串后移2位 O(N^2)
方法二：O(N) TIME O(N) SPACE WORST CASE (All space多分配两倍内存)

3. 从尾到头打印链表

输入一个链表，按链表值从尾到头的顺序返回一个ArrayList。 leetcode 206
方法一：反转链表之后把值遍历加进arraylist O(N) TIME O(1) SPACE（如果可以inplace modify linkedlist）

方法二：使用stack来反转值的顺序 因为stack是顺序储存先进先出 可以起到反转顺序的作用 O(N) TIME O(N) SPACE

4. 重建二叉树

输入某二叉树的前序遍历和中序遍历的结果，请重建出该二叉树。假设输入的前序遍历和中序遍历的结果中都不含重复的数字。例如输入前序遍历序列{1,2,4,7,3,5,6,8}和中序遍历序列{4,7,2,1,5,3,8,6}，则重建二叉树并返回。

首先我们知道preorder[0]是root 通过找到他在inorder中的位置我们可以知道左子树右子树的节点数量和具体节点
对于每一个子树的节点列表 preorder’’[0]仍是子树的根这样递归的进行树的重建。 但是对于非平衡树 或者极端情况一条链表 在inorder里搜索跟的位置要时间复杂度O(n^2) 并且取决于怎样split我们甚至可能需要空间复杂度O(n^2) 我们可以使用map来降低时间空间复杂度 但仍要O(n)
若不使用map我们也可以达到时间O(n)空间O(n)我们手动设置递归终止条件：
每个stack frame上我们使用当前子树根节点的值作为其左子树的递归终止条件 右子树使用当前的stop值作为递归终止条件
Instead of finding a subtree root in inorder, splitting the arrays into parts and recursing on them, just recurse on the full remaining arrays and stop when you come across the subtree root in inorder. Each recursive call gets told where to stop, and it tells its subcalls where to stop. It gives its own root value as stopper to its left subcall and its parent‘s stopper as stopper to its right subcall.
简单的observation:

in = 4251637
pre = 1245367
post = 4526731

LL左子树的左孩子终止条件：其父节点的值
LR左子树的右孩子终止条件：其爷爷节点的值
RL右子树的左孩子终止条件：其父节点的值
RR右子树的右孩子：最右边的branch 根节点的stop
保持两个指针pre in pre>=preorder.length控制最右边branch递归 inorder[in]==stop控制常规递归 每一次stop就等于遍历完了一个左/右branch
preorder = [3,9,20,15,7]
inorder = [9,3,15,20,7]
递归树：

leetcode106 从inorder和postorder重建二叉树：
注意通过之前题目不难发现
preorder = 从左到右的dfs
postorder_reversed = 从右到左到dfs
那么preorder+inorder 等价于reversed postorder+reversed inorder+right to left build tree
所以我们继续之前解法 唯一不同是将postorder及inorder反转 然后将recursion从左到右改为从右到左, right stop = root val left stop = stop. 仍然是O(n) space and time.

6. 旋转数组最小值

把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。 输入一个非减排序的数组的一个旋转，输出旋转数组的最小元素。 例如数组{3,4,5,1,2}为{1,2,3,4,5}的一个旋转，该数组的最小值为1。 NOTE：给出的所有元素都大于0，若数组大小为0，请返回0。

O(N) O(1) Brute Force

O(logN) O(1) Binary Search

leetcode509 现在要求输入一个整数n，请你输出斐波那契数列的第n项（从0开始，第0项为0）。n<=39
方法一：O(1) O(N) Hard code all value in an array

8. 跳台阶

leetcode70 一次可以跳上1级台阶，也可以跳上2级。求跳上一个n级的台阶总共有多少种跳法（先后次序不同算不同的结果）。
7,8,leetcode 91-decode ways, 62-unique paths均是相同的问题. 简单的dp递推 n级的解=n-1级的解+n-2级的解

9. 变态跳台阶

一次可以跳上1级台阶，也可以跳上2级……也可以跳上n级。求跳上一个n级的台阶总共有多少种跳法。 本题意思是可以跳任何数量为1…n的步数
n = 0: F(0) = 0;
n = 1: F(1) = 1;
n = 2: 两种跳法 第一次跳一步或者两步 F(2) = F(1) + F(0) = 2;
n = 3: 第一次跳1，2，3步 F(3) = F(2) + F(1) + F(0) = 4;
n = n: 第一次跳1，2，3，4，5，…，n步
F(n) = F(n-1)+F(n-2)+F(n-3)+…+F(n-n)=F(0)+F(1)+F(2)+…+F(n-1)
Plugin n - 1: F(n-1)=F(0)+F(1)+F(2)+…+F(n-2)
Fib(n) - Fib(n-1) = Fib(n-1), Fib(n) = 2Fib(n-1)

10. 矩形覆盖

我们可以用21的小矩形横着或者竖着去覆盖更大的矩形。请问用n个21的小矩形无重叠地覆盖一个2n的大矩形，总共有多少种方法？
仍然是一个DP问题

如图所示若右边28大矩形先被覆盖了竖着的最左边两个格子 则还有27个需要被覆盖。 若先被横着覆盖了左上角两个格子，则最左边22正方形只有1种覆盖方法，同时剩余问题规模2*6
F(0) = 0, F(1) = 1, F(2) = 2
N>=3 F(N)=F(N-1) + F(N-2)

11. 二进制中1的个数

leetcode 191 Hamming weight
方法1: O(32) = O(1)
枚举每一位 每一轮右移1位得到lsb 等于1则count++ 然后rotate right 进行下一轮

注意>> signed right shift >>>unsigned right shift
方法2: O(#1) = O(1)
只数有多少个1 通过x=x&(x-1)的方式每一轮删掉最后一个1直到x=0停止计数

64. 滑动窗口的最大值

sliding window of size k in array size of N 总共有n-k+1个！

滑动窗口最大值leetcode239
brute force approach: o((n-k+1)*k) o(1)space

Using deque : o(n) o(k)
使用双端队列来保存最大元素的index
遍历所有nums中元素
第1个while：删掉所有不在sliding window里的元素（被划过去的）
第二个while：从后往前删掉deque里所有比当前元素小的
在deque里添加当前index
只有从i=k-1起才往result里加值
